大致需求：
1、商品档案管理
创建产品档案，系统自动生成商品编号，上传产品图片，录入商品售价。同款商品可多次分批次入库，每一批次单独记录单件制作成本，库存自动合计。货品出库核算成本采用先进先出，可查看各批次入库记录及剩余库存。

2、日常销售开单
挑选商品生成销售清单，单品可填写实际折后成交价格。确认成交后，现货库存自动扣减，生成销售记录，系统自动核算本次销售额与利润，按日、月、年汇总。

3、寄售铺货管理
单独设立寄售出库单据，填写铺货数量、合作方、约定结算价。货品从现货库存转出，归入寄售在外库存，暂不计入营收利润。后续可对售出部分做回款结算，转为正式销售；未售出货品可操作退回，重新回到现货库存。

4、台账与数据统计
分开保存日常销售台账、寄售台账，可按日期查询订单明细。统计板块区分现货盈利和寄售结算盈利，分别展示现有现货成本、寄售货品成本、待回款总额，自动汇总每日、每月、全年经营数据。

详细设计：
用户为艺术手工制作个体户，制作一款适用手机网页的应用，用我的的github发布网页，应用主要为仓库记账使用。
我想制作主要分为四个模块，仓库，记事，记账，我。
一、首页仓库模块
顶端设置搜索栏，可以搜索自己仓库内已入库的商品，搜索栏旁边有个放大镜，点击即可搜索相关商品。搜索栏也可以根据字符，弹出相关延伸搜索名称，用户点击即可。
下面设置长方形按钮，为“录入”功能，点击后，弹出两个按钮，一个为新商品，一个为新分类，点击新商品，用户可以上传商品名称，图片，成本价，售价，数量，状态（正在售出，未售出，已售出，待上架），分类区间（可以设置为空）。点击新分类，用户可以增加商品分类区间，自定义名称
再下面设置四个子模块用来显示仓库：未售出，已售出，待上架，正在售出。页面下面默认显示最近入库的商品，排序按更新时间由近到远，同时设计一个栏来进行筛选，价格区间，价格升序或降序，名称搜索（功能和顶端搜索栏一致），分类区间，入库时间（由远到近，由近到远）。优先根据商品的状态，然后再根据商品的分类，来显示商品。
再下面采用紧凑的类似淘宝页面显示商品，大图下面是，名称，成本，售价，状态，上架时间（若未上架，显示未上架），售出时间（若未售出，显示待售），同时每个商品显示页面右上角有个管理按钮，点击该按钮可以重新设置其以上说的商品属性，同时增加一个删除功能，选择商品数量进行删除。
二、记事模块
顶端一个小模块，今日小目标，下面用户可以添加今日目标，今日目标可以设置名称，时间（可以无），地点（可以无），添加一个代办项，下面就弹出一个代办项，每个代表项右边有完成和取消按钮
下面一个小模块，是长期目标，点击便可以弹出来所有长期目标，默认是收起来的，把添加长期目标按钮设置在此小模块的最下面，每个长期目标设置名称，时间起始区间，倒计时（天数）。同时页面右上角设置一个日历按钮，点击日历，弹出来日历，点击时间可以跳转到当日查看待办事项和完成情况
再下面一个板块设置今日正售，下面会默认显示状态为正在售出的商品，右边一个加号，可以添加仓库里未售出的商品。展示商品的方式类似仓库模块的展示方式，默认按上架时间来显示正在售出的商品，同时也有价格筛选，名称搜索，分类筛选。每个商品仅显示图片，名称，成本价，售价，数量，
在此模块每个商品右上角页面增加一个售出按钮和管理按钮，点击售出后，弹出实际售出价（元）和数量，用户输入后，点确定，商品自动减数量，如果售空，则显示数量0。点击管理按钮可以让商品进行状态更改为未售出状态和下架状态（下架则商品的仓库状态变为待上架）
此大模块的商品状态，数量，和仓库都是相关联的数据库。
三、记账模块，主要用于统计用户今日，最近一周，最近一月，最近一年，过往全部，售出哪些商品，成本多少，售出多少，利润多少。
同时用户可以添加贩卖商品外的其他收入和开支，比如（上课，设计等等，用户自定义收入和支出选项）
设置日历按钮，点击日历，可以查看当日的收入，开支情况，以及售出了哪些货物
四，我模块
设置日历功能和日记功能，
日历在顶端，显示年月日，日历按钮，在右边
日历功能性在下面
可以撰写当日心得，上床图片。
点击日历按钮，可以跳转到指定时间查看当日日历
